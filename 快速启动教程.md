# 行动安全守护系统 — 快速启动教程

面向零基础用户。跟着做，10 分钟内在自己电脑上跑起来。

---

## 第一步：安装 Node.js

Node.js 是运行这个项目必需的环境，就像手机需要安卓/iOS 系统一样。

### 1.1 下载

打开浏览器，访问：

```
https://nodejs.org
```

点击左边的 **LTS** 按钮下载（版本号可能比截图高，正常）。

下载完成后会得到一个 `node-vXX.XX.X-x64.msi` 文件。

### 1.2 安装

双击下载的 `.msi` 文件：

1. 点击 **Next**
2. 勾选 **「I accept the terms」** → Next
3. 安装路径保持默认（`C:\Program Files\nodejs\`）→ Next
4. 不需要勾选额外工具 → Next
5. 点击 **Install**
6. 等待进度条走完 → Finish

### 1.3 验证安装是否成功

按键盘 `Win + R`，输入 `cmd`，回车。会弹出一个黑色窗口。

在黑色窗口里输入：

```bash
node -v
```

如果看到类似 `v18.XX.X` 或 `v20.XX.X` 的版本号，说明安装成功。

继续输入：

```bash
npm -v
```

看到类似 `10.X.X` 的版本号，说明 npm 也装好了。

> 如果提示"不是内部或外部命令"，说明安装未成功。重新双击 .msi 文件，选择 Repair 修复即可。

## 第二步：获取项目代码

### 方式 A：已有项目文件夹

如果你已经有 `modell-2` 文件夹，跳过这一步，直接进入第三步。

### 方式 B：从 Git 拉取（如果你用过 Git）

```bash
git clone <项目地址>
cd modell-2
```

## 第三步：安装项目依赖

### 3.1 进入 web 目录

在资源管理器中，找到你的 `modell-2` 文件夹，打开里面的 `web` 文件夹。

点击地址栏，输入 `cmd`，回车：

> 黑色窗口会在 `modell-2\web` 目录下打开。

### 3.2 安装依赖包

在黑色窗口中输入下面这行命令，然后回车：

```bash
npm install
```

你会看到大量文字滚动，持续 1-2 分钟。最后会显示类似：

```
added XXX packages in XXs
```

这就说明安装完成了。

> 如果卡住不动超过 5 分钟，按 `Ctrl + C` 终止，重新输入 `npm install` 再试一次。

---

## 第四步：启动项目

### 4.1 构建前端

在同一个黑色窗口中输入：

```bash
npm run build
```

等待约 30 秒，看到 `✓ built in XXXms` 就说明构建成功。

### 4.2 启动后端服务

输入：

```bash
npm run server
```

看到下面三行字就说明启动成功：

```
  服务已启动: http://localhost:3001
  前端页面: http://localhost:3001/
  API 健康: http://localhost:3001/api/health
```

## 第六步：关闭 / 重新启动

### 关闭

在黑色窗口中按 `Ctrl + C`，服务就停了。关闭窗口也行。

### 重新启动

以后每次想用，重复下面两步就够了：

```bash
# 在 modell-2\web 目录下打开 cmd

npm run build        # 第1步：构建（如果没改过代码，第二次起可以跳过）
npm run server       # 第2步：启动服务
```

然后浏览器打开 `http://localhost:3001`。

## 常见问题

### Q: 提示 "npm 不是内部或外部命令"

Node.js 没装好。回到第一步，重新安装 Node.js。安装时确保「Add to PATH」被勾选。

### Q: 提示 "Error: Cannot find module xxx"

依赖没装全。输入：
```bash
npm install
```
等待完成后再试。

### Q: 浏览器显示「无法访问此网站」

后端服务没启动。确认黑色窗口里能看到「服务已启动」字样。如果窗口被关了，重新执行第四步。

### Q: 端口 3001 被占用

输入下面命令换一个端口：
```bash
set PORT=3002 && npm run server
```
然后浏览器访问 `http://localhost:3002`。

### Q: 登录后页面空白或报错

可能是前端没构建或者构建版本旧。重新执行：
```bash
npm run build
npm run server
```
然后刷新浏览器（`Ctrl + F5` 强制刷新）。

### Q: 注册时验证码在哪看？

Mock 模式下，点击"发送验证码"后，6 位验证码会打印在运行 `npm run server` 的黑色命令行窗口中。找到 `[Email Mock]` 开头的行，里面的 6 位数字就是验证码。

### Q: 短信验证码收不到？

默认 MCK 模式，短信验证码也会打印在命令行窗口中（`[SMS Mock]` 开头）。如需真实发送，参见上方「启用真实邮件和短信」章节配置阿里云短信。

### Q: 白色窗口闪一下就没了

地址栏输入 `cmd` 而不是直接双击程序。确保黑色窗口显示的是 `\modell-2\web>` 路径。

### Q: 想在其他设备（手机/平板）上访问

系统已内置移动端适配，手机浏览器可直接使用。访问步骤：

1. 查看电脑 IP：Win + R → 输入 `cmd` → 输入 `ipconfig` → 找到「IPv4 地址」如 `192.168.1.165`
2. 手机浏览器输入 `http://192.168.1.165:3001`（用你的实际 IP）
3. 手机和电脑必须连同一个 Wi-Fi

> 手机端会自动切换为抽屉式导航菜单，表格支持横向滑动，所有页面均已适配小屏幕。
>
> 如果无法访问，检查 Windows 防火墙是否拦截。在「Windows 安全中心 → 防火墙 → 允许应用通过防火墙」中放行 Node.js。

## 启用真实邮件和短信（可选）

默认情况下，验证码会打印在黑色命令行窗口中（Mock 模式），方便开发测试。

如果需要真实发送邮件和短信，需配置环境变量。

### 邮件发送策略

系统采用三级降级：**Resend API → QQ邮箱 SMTP → Mock**。配置其中一种即可。

### 方式 A：Resend（推荐）

支持向任意邮箱发送，免费 100 封/天，不进垃圾箱。

1. 注册 [Resend](https://resend.com) 账号
2. 获取 API Key（`re_` 开头）
3. 启动服务前设置：

```bash
# Windows
set RESEND_API_KEY=re_xxxxxxxx
npm run server

# Linux / macOS
export RESEND_API_KEY="re_xxxxxxxx"
```

> 测试模式下仅发给自己的邮箱。验证域名后可发给任意用户，详见 Resend 控制台 → Domains。

### 方式 B：QQ 邮箱 SMTP（备用）

```bash
set SMTP_USER=你的QQ号@qq.com
set SMTP_PASS=你的授权码
npm run server
```

1. 登录 QQ 邮箱 → 设置 → 账户 → POP3/SMTP 服务 → 开启
2. 获取授权码（一串字母，不是 QQ 密码）
3. 每天限额约 500 封，仅能以自己的 QQ 邮箱身份发送

### 启用阿里云短信

1. 登录阿里云控制台 → 开通短信服务 → 申请签名和模板
2. 获取 AccessKey
3. 设置环境变量：

```bash
set SMS_PROVIDER=alicloud
set ALICLOUD_ACCESS_KEY_ID=你的AccessKey
set ALICLOUD_ACCESS_KEY_SECRET=你的Secret
set ALICLOUD_SMS_SIGN_NAME=你的短信签名
set ALICLOUD_SMS_TEMPLATE_CODE=你的模板ID
npm run server
```

> 不配置以上变量不影响系统正常运行，验证码会自动降级为 Mock 模式。

## 部署到公网（Serveo 固定域名）

Serveo 是一个免费的 SSH 隧道服务，能把你的本地服务暴露到公网，得到一个别人可以访问的固定网址。无需买服务器、无需备案。

### 前置条件

确认电脑有 SSH。打开 cmd，输入：

```bash
ssh -V
```

看到版本号说明 OK。Windows 10/11 自带 SSH，无需额外安装。

### Step 1：生成 SSH 密钥

在 cmd 中输入下面命令，生成一把专门给 Serveo 用的密钥：

```bash
ssh-keygen -t rsa -b 4096 -f %USERPROFILE%\.ssh\id_rsa_serveo -N ""
```

> 这会在 `C:\Users\你的用户名\.ssh\` 目录下生成两个文件：`id_rsa_serveo`（私钥）和 `id_rsa_serveo.pub`（公钥）。

### Step 2：查看并复制公钥

```bash
type %USERPROFILE%\.ssh\id_rsa_serveo.pub
```

输出的一长串文字（以 `ssh-rsa` 开头），全部选中复制。这就是你的公钥。

### Step 3：注册 Serveo 账号

1. 浏览器打开 `https://console.serveo.net`
2. 用 Google 或 GitHub 账号登录（没有的话先注册一个 GitHub 账号）
3. 登录后，左侧菜单点击 **SSH Keys**
4. 点击 **Add SSH Key**，把刚才复制的公钥粘贴进去
5. 点击保存

### Step 4：预约固定子域名

1. 左侧菜单点击 **Domains**
2. 点击 **Add Domain**
3. 在 Hostname 输入框输入你想要的子域名，例如 `mobility-guardian`
4. HTTP Forwarding Type 选择 **SSH**
5. 点击确认

> 子域名只能用小写英文字母 + 数字 + 连字符。如果提示已被占用，换一个名字。

通过后你的固定域名就是：`https://你的名字.serveousercontent.com`

### Step 5：启动

**需要同时开两个 cmd 窗口。**

**窗口 1 — 启动后端服务：**

```bash
cd 你的项目路径\web
npm run build
npm run server
```

看到「服务已启动: http://localhost:3001」说明 OK。

**窗口 2 — 启动公网隧道：**

```bash
ssh -i %USERPROFILE%\.ssh\id_rsa_serveo -R 你的名字:80:localhost:3001 serveo.net
```

> **`-i` 参数不能省略！** 它指定使用你注册的那把密钥。不用 `-i` 的话 Serveo 认不出你，会给一个随机域名。

看到 `Forwarding HTTP traffic from https://你的名字.serveousercontent.com` 就成功了。

### Step 6：验证

用手机或别人的电脑打开 `https://你的名字.serveousercontent.com`，应该能看到登录页面。

> 免费版首次访问会显示一个提醒页面，点击按钮即可进入网站。

### 固定域名总结

| 动作 | 命令 |
|------|------|
| 启动服务 | `npm run build && npm run server` |
| 启动隧道 | `ssh -i %USERPROFILE%\.ssh\id_rsa_serveo -R 你的名字:80:localhost:3001 serveo.net` |
| 关闭 | 两个窗口分别按 `Ctrl + C` |

以后每次启动，域名都是固定的，不会变。需要保持两个窗口都开着。

### 常见问题

**Q: 提示 "To request a particular subdomain, you first need to register your SSH public key"**

说明 Serveo 没认出你的密钥。检查两点：
1. 是否用了 `-i` 参数指定密钥路径？直接 `ssh serveo.net` 会用默认密钥，必须加 `-i`
2. 公钥是否已添加到 Serveo 控制台的 SSH Keys 页面？

**Q: 手机能访问，外面的人访问不了**

确认：
1. 两个 cmd 窗口都开着，没有报错
2. 后端服务窗口显示「服务已启动」
3. 隧道窗口显示 `Forwarding HTTP traffic from https://...`
4. 防火墙没拦截（一般不会，SSH 是出站连接）

**Q: 免费版有什么限制？**

- 每次访问前会显示一个提醒页面（访客点一下即可进入）
- 长时间无流量隧道可能断开，重新执行 Step 5 即可
