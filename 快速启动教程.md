# 第一部分：网页端启动

## 1.1 安装 Node.js

Node.js 是运行这个项目必需的环境。

1. 浏览器打开 `https://nodejs.org`
2. 点击左边的 **LTS** 按钮下载
3. 双击下载的 `.msi` 文件，一路 Next 安装
4. 按 `Win + R`，输入 `cmd`，回车，输入 `node -v`

看到 `v18.XX.X` 或 `v20.XX.X` 说明安装成功。

## 1.2 安装依赖

在资源管理器中打开 `web/web` 文件夹，点击地址栏输入 `cmd` 回车，然后：

```bash
npm install
```

等待 1-2 分钟，看到 `added XXX packages` 即完成。

## 1.3 构建并启动

```bash
npm run build
npm run server
```

看到以下输出说明启动成功：

```
  服务已启动: http://localhost:3001
  前端页面: http://localhost:3001/
  API 健康: http://localhost:3001/api/health
  WebSocket: ws://localhost:3001
```

浏览器打开 `http://localhost:3001`，用 `admin / admin123` 登录。

## 1.4 默认账户

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | admin123 | 管理员 |
| zhangsan | demo123 | 家属 |
| lihua | demo123 | 家属 |
| laowang | demo123 | 被监护人 |

> 首次启动自动创建 admin。其他演示账户可自行注册，或运行 `node server/seed.cjs` 一键生成 zhangsan / lihua / laowang 并附带历史数据和监护人关系。注册时验证码打印在命令行窗口。

## 1.5 开发模式（改代码时用）

```bash
# 终端 1：后端
npm run server

# 终端 2：前端（改代码自动刷新）
npm run dev
```

浏览器打开 `http://localhost:5174`。

## 1.6 手机浏览器访问（同 WiFi）

1. 查看电脑 IP：`Win + R` → `cmd` → `ipconfig` → 找 IPv4 地址，如 `192.168.1.165`
2. 手机浏览器输入 `http://192.168.1.165:3001`
3. 手机和电脑必须连同一个 Wi-Fi

> 如果无法访问，检查 Windows 防火墙：安全中心 → 防火墙 → 允许应用通过防火墙 → 放行 Node.js。

# 第二部分：移动端 APK 打包

## 2.1 前置条件

需要安装以下工具（已有可跳过）：

- **Java 17**：推荐 [Zulu JDK 17](https://www.azul.com/downloads/#zulu)
- **Android SDK**：安装 Android Studio 时自带，或单独安装

验证 Java：

```bash
java -version
# 应显示 17.x
```

## 2.2 安装依赖并构建前端

在 `app` 目录下打开 cmd：

```bash
npm install
npm run build
```

## 2.3 打包 APK

```bash
cd android
./gradlew assembleDebug --init-script init.gradle
```

看到 `BUILD SUCCESSFUL` 即成功。

APK 位置：`app/android/app/build/outputs/apk/debug/app-debug.apk`

## 2.4 配置后端地址

打包前可修改 `app/src/api/index.ts` 第 10 行：

```ts
const SERVER_URL = localStorage.getItem('server_url') || 'http://你的电脑IP:3001';
```

改成你电脑的实际 IP，这样 APK 装到手机上就能连上后端。

## 2.5 安装到手机

1. 把 APK 传到手机
2. 点击安装（需允许"未知来源"安装）
3. 打开 App，确保手机和电脑连同一 WiFi

# 第三部分：公网访问（固定域名）

Serveo 是免费 SSH 隧道服务，能把本地服务暴露到公网，获得一个固定的 HTTPS 地址。**配置一次后域名永久不变**。

## 3.1 生成 SSH 密钥（只需一次）

```bash
ssh-keygen -t rsa -b 4096 -f %USERPROFILE%\.ssh\id_rsa_serveo -N ""
```

## 3.2 注册 Serveo 并预约域名（只需一次）

1. 复制公钥：`type %USERPROFILE%\.ssh\id_rsa_serveo.pub`
2. 浏览器打开 `https://console.serveo.net`，用 GitHub/Google 登录
3. 左侧 **SSH Keys** → Add SSH Key → 粘贴公钥
4. 左侧 **Domains** → Add Domain → 输入子域名（如 `mobility-guardian`）
5. HTTP Forwarding Type 选择 **SSH**

通过后你的固定域名：`https://你的名字.serveousercontent.com`

## 3.3 每次启动（开两个 cmd 窗口）

**窗口 1 — 后端：**

```bash
cd web/web
npm run build
npm run server
```

**窗口 2 — 公网隧道：**

```bash
ssh -i %USERPROFILE%\.ssh\id_rsa_serveo -R 你的名字:80:localhost:3001 serveo.net
```

看到 `Forwarding HTTP traffic from https://...` 即成功。

> 免费版首次访问有提醒页，点击按钮即可进入。长时间无流量隧道可能断开，重新执行窗口 2 命令即可。

# 第四部分：设备模拟器

在没有真实嵌入式设备时，用模拟器产生测试数据：

```bash
cd web/web
node server/simulator.cjs
```

模拟器会持续运行，每 2 秒上报一次数据，包括：
- 随机切换站立/行走状态
- 累加步数、消耗电量
- 约每 100 秒触发一次跌倒（自动生成告警）

打开数据看板页面，可以看到数据实时变化。

# 第六部分：体验在线聊天（类似微信的实时沟通）

系统实现了完整的"好友申请 → 同意 → 实时聊天"流程。

### 6.1 完整流程

1. **注册两个账号**（或用种子数据中的 zhangsan / lihua / laowang）
2. **登录 A 账号** → 点击「监护人管理」→「搜索用户」
3. **输入 B 的用户名** → 点击「发送监护申请」
4. **切换到 B 账号** → 右上角红点通知 → 点进「监护人管理」→「收到的申请」→ 点「同意」
5. **A 收到申请通过通知** → 双方联系人列表出现对方
6. **点击「消息」菜单** → 选择对方 → 发送消息 → 对方实时收到

### 6.2 注意事项

- WebSocket 实时推送需要后端运行中。开发模式（`npm run dev`）下，前端在 5174 端口，WebSocket 连 3001 端口的后端，已自动配置。
- 消息同时写入数据库，聊天记录不会丢失。
- 移动端 APK 中的聊天功能需要手机能访问后端（同 WiFi 或公网域名）。

## 常见问题

### Q: 提示 "npm 不是内部或外部命令"
Node.js 没装好。回到 1.1 重新安装。

### Q: 端口 3001 被占用
```bash
set PORT=3002 && npm run server
```
浏览器访问 `http://localhost:3002`。

### Q: 注册时验证码在哪看？
Mock 模式下验证码打印在运行 `npm run server` 的黑色命令行窗口中。

### Q: 登录后页面空白或报错
重新构建：
```bash
npm run build
npm run server
```
然后浏览器 `Ctrl + F5` 强制刷新。

### Q: APK 打包报错 "Failed to find Platform SDK"
本地 Android SDK 缺少对应平台。已在 `init.gradle` 中配置了兼容方案，确保使用 `--init-script init.gradle` 参数。

### Q: APK 打包报错 "无效的源发行版：21"
Java 版本低于 21。`init.gradle` 已自动降级到 Java 17，确保本地安装了 JDK 17。

### Q: Serveo 隧道连不上
国内网络可能不稳定，多试几次或换个网络环境。

### Q: 为什么消息页面看不到监护人？
联系人列表同时包含：1）与你互发过消息的人 2）你的监护人和被监护人。如果刚添加监护人还没显示，刷新页面即可。

### Q: 为什么我发的消息对方收不到？
检查两点：1）后端是否正在运行 2）前端的 WebSocket 地址是否正确（开发模式下 `useSocket.ts` 中 `SOCKET_URL` 应指向 `http://localhost:3001`）。

### Q: 申请监护人的流程是怎样的？
不是单方面添加，而是"申请制"：搜索用户名 → 发送申请 → 对方在「监护人管理」页面看到申请 → 同意或拒绝。同意后双方互为监护人，可以开始聊天。
