# 第一部分：网页端启动

## 1.1 方式 A：直接访问云端（推荐，无需任何操作）

浏览器打开：

```
https://modell-2-production.up.railway.app
```

注册账号，立即使用所有功能。

## 1.2 方式 B：本地运行（需要 Node.js）

### 安装 Node.js

1. 浏览器打开 `https://nodejs.org`
2. 点击左边的 **LTS** 按钮下载
3. 双击 `.msi` 文件，一路 Next 安装
4. `Win + R` → `cmd` → 输入 `node -v`，看到版本号即成功

### 安装依赖

在资源管理器中打开 `web/web` 文件夹，地址栏输入 `cmd` 回车：

```bash
npm install
```

### 构建并启动

```bash
npm run build
npm run server
```

看到以下输出说明启动成功：

```
  服务已启动: http://localhost:3001
  前端页面: http://localhost:3001/
  API 健康: http://localhost:3001/api/health
  WebSocket: ws://localhost:3001
```

浏览器打开 `http://localhost:3001`。

### 开发模式（改代码时用）

```bash
# 终端 1：后端
npm run server

# 终端 2：前端（Vite HMR，端口 5174）
npm run dev
```

# 第二部分：移动端 APK 打包

## 2.1 直接安装（推荐）

将 `app/android/app/build/outputs/apk/debug/app-debug.apk` 传到手机安装即用。APK 已预配置 Railway 云端地址，**打开就能用，零配置**。

## 2.2 手动打包

### 前置条件
- Java 17（Zulu JDK）
- Android SDK（Build Tools 36.1.0+，Platform android-36）

### 打包步骤

```bash
cd app
npm install
npm run build

rm -rf android/app/build
cp -r dist android/app/src/main/assets/public
cd android
./gradlew assembleDebug --init-script init.gradle --offline
```

> 首次打包不要用 `--offline`，让 Gradle 下载依赖。之后每次改前端代码后重新构建用上述命令。

APK 位置：`app/android/app/build/outputs/apk/debug/app-debug.apk`

# 第三部分：云端部署（Railway）

项目已部署在 Railway，推送 GitHub 即自动更新。

**已部署地址**：`https://modell-2-production.up.railway.app`

## 部署原理

```
GitHub push → Railway 自动检测 → npm install + npm run build → node server/index.cjs
```

每次 push 到 GitHub 的 `start` 或 `main` 分支，Railway 自动重新部署，约 3-5 分钟完成。

# 第四部分：设备模拟器

在没有真实嵌入式设备时，用模拟器产生测试数据（仅本地运行）：

```bash
cd web/web
node server/simulator.cjs
```

模拟器会：
- 每 5 秒上报一次数据
- 随机切换站立/行走状态
- 约每 250 秒触发一次跌倒（自动生成告警）

打开数据看板页面，可以看到数据实时变化。

# 第六部分：体验在线聊天（类似微信的实时沟通）

## 6.1 完整流程

1. **注册两个账号**（或用种子数据中的 zhangsan / lihua）
2. **登录 A 账号** → 点击「监护人管理」→「搜索用户」
3. **输入 B 的用户名/邮箱/手机号** → 点击「发送监护申请」
4. **切换到 B 账号** → 右上角红点通知 → 点进「监护人管理」→「收到的申请」→ 点「同意」
5. **A 收到申请通过通知** → 双方联系人列表出现对方
6. **点击「消息」菜单** → 选择对方 → 发送消息 → 对方实时收到

## 6.2 注意事项

- 消息同时写入数据库，聊天记录不会丢失
- 网页端和移动端均可实时收发消息
- 未读消息会在侧边栏显示红点角标

## 常见问题

### Q: 提示 "npm 不是内部或外部命令"
Node.js 没装好。回到 1.2 重新安装。

### Q: 端口 3001 被占用
```bash
set PORT=3002 && npm run server
```
浏览器访问 `http://localhost:3002`。

### Q: 注册时验证码在哪看？
Mock 模式下，点击"发送验证码"后，6 位验证码会**直接显示在页面上**（绿色大字），不用看任何其他地方。

### Q: 登录后页面空白或报错
重新构建：`npm run build && npm run server`，然后 `Ctrl + F5` 强制刷新。

### Q: APK 打包报错 "Failed to find Platform SDK"
本地 Android SDK 缺少对应平台。确保使用 `--init-script init.gradle` 参数。

### Q: APK 打包报错 "无效的源发行版：21"
Java 版本低于 21。`init.gradle` 已自动降级到 Java 17，确保本地安装了 JDK 17。

### Q: 申请监护人的流程是怎样的？
"申请制"：搜索用户名/邮箱/手机号 → 发送申请 → 对方在「监护人管理」看到申请 → 同意或拒绝。同意后双方互为监护人，可以开始聊天。

### Q: 云端部署的数据库会丢失吗？
Railway 免费实例每次重启/部署会重置磁盘，数据库会清空。如需持久化，在 Railway 控制台添加 Volume，挂载到 `server/` 目录。
